## 1. 选择原则（超越“好看”）

- 结构优先于视觉：信息层级、导航、PDP/Collection 模块是否符合业务。
- 扩展性：Section/Block 设计、与 Metafields/Metaobjects 兼容、可否做多市场。
- 性能：LCP/CLS 基线、资源体积、第三方脚本治理支持。
- 长期成本：更新频率、支持质量、与常用 App 兼容性。

## 2. 评估维度

- 信息架构：是否有清晰的 Collection/PDP 模板、导航、过滤。
- 可配置性：无需写代码即可调整模块；是否支持自定义模板。
- 多语言/多货币：内置支持还是需插件。
- 性能与 SEO：首屏加载、结构化数据、可访问性。
- 售后与社区：更新日志、Issue 响应、文档质量。
- 兼容性：与搜索/推荐/翻译/分析类 App 的已知兼容情况。

## 3. 选型 Checklist

- [ ] 主题 Demo 的信息结构符合你的品类（非仅视觉）。
- [ ] 支持关键模板：PDP（含变体/媒体）、Collection（过滤/排序）、Blog/Content（如需）。
- [ ] Section/Block 丰富且与 Metafields/Metaobjects 兼容，可少写代码完成配置。
- [ ] 性能测试通过：首屏体积、LCP/CLS 可接受；无过度脚本。
- [ ] 多语言/多货币需求有内置方案或简单集成路径。
- [ ] 有更新与支持承诺，历史更新频率可信。
- [ ] 与计划使用的关键 App（搜索/推荐/翻译/分析）无已知冲突。
- [ ] 试用期内可快速搭建 PoC，验证 PDP/Cart/Checkout 体验。
